XPresso C.O.F.F.E.E.

Have you ever wished you could load a midi file and your musical instrument would start to play it (in C4D) ? It’s just what I wanted to do. With the Sound Effector you can load a .wav or .aif file. But as there are overtones (harmonic) the result is not exactly what a real musician would play even if you set up the filters yourself.

I know. I did it for a 88 note piano. Setting up 88 filters and fixing 88 bandwidths.


The first problem is to load the .mid file. Parsing a .mid file is not too difficult. But it’s not plain text. So if you don’t eat hexadecimal for breakfast, the first step would be to convert your .mid file into a text file. Luckily there are plenty of freewares on Windows and Mac that do just that. The big disadvantage of going that road is that your parser is tied to the converter that you are using, as their outputs differ.


A .mid file may look like this in text format

MTrk

To load I’ve found two ways


  1. -The no XPresso way


C4D has its own format for writing and reading (Load Data & Read Data)

What’s the difference between C4D format and plain text format. Just a short header

        


What’s that ? OK in hex



Up to the first red line it identifies the file as a C4D file. It’s always the same.

After, 0C in hexadecimal means 12 in decimal. It mean that there are 12 bytes of data.

Two bytes per characters. So 6 characters. So if you want to use this method , you must add manually that information before C4D can read it.


How do you read it once this is done ? Easy.

You “Add User Data” with a Data type string

and for Interface “Multi-line String”.


Once this is done you just right-click on

ºData and scroll down to Load Data


That’s it.
